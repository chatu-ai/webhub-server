## Overview

WebHub Channel Service supports two primary message flows:

| Direction | Type | Description | Endpoint |
|-----------|------|-------------|----------|
| **Inbound** | `InboundMessage` | Messages received from the channel | `POST /api/channel/messages` |
| **Outbound** | `OutboundMessage` | Messages sent to the frontend via the hub | `POST /api/channel/messages` |

---

## Core Message Types

### Inbound Message

Format for messages received from an external channel.

#### TypeScript Definition

```typescript
interface InboundMessage {
  /** Unique message identifier */
  id: string;

  /** Channel ID */
  channelId: string;

  /** Message timestamp (Unix milliseconds) */
  timestamp: number;

  /** Sender information */
  sender: {
    id: string;
    name?: string;
    avatar?: string;
  };

  /** Message content */
  content: {
    text: string;
    format?: 'plain' | 'markdown' | 'html';
  };

  /** Media attachments (optional) */
  media?: Media[];

  /** ID of the message being replied to (optional) */
  replyTo?: string;
}
```

#### Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | ✅ | Unique message identifier, generated by the sender |
| `channelId` | `string` | ✅ | Channel unique identifier |
| `timestamp` | `number` | ✅ | Message send time (Unix timestamp, milliseconds) |
| `sender` | `object` | ✅ | Sender information |
| `sender.id` | `string` | ✅ | Sender unique identifier |
| `sender.name` | `string` | ❌ | Sender display name |
| `sender.avatar` | `string` | ❌ | Sender avatar URL |
| `content` | `object` | ✅ | Message content |
| `content.text` | `string` | ✅ | Text content of the message |
| `content.format` | `string` | ❌ | Text format: `plain`, `markdown`, or `html`. Defaults to `plain` |
| `media` | `array` | ❌ | Array of media attachments (see [Media](#media)) |
| `replyTo` | `string` | ❌ | ID of the message being replied to |

### Outbound Message

Format for messages sent to the frontend via the hub.

#### TypeScript Definition

```typescript
interface OutboundMessage {
  /** Unique message identifier (optional; auto-generated if omitted) */
  messageId?: string;

  /** Channel ID */
  channelId: string;

  /** Message target */
  target: {
    type: 'user' | 'group' | 'channel';
    id: string;
    name?: string;
  };

  /** Message content */
  content: {
    text: string;
    format?: 'plain' | 'markdown' | 'html';
  };

  /** Media attachments (optional) */
  media?: Media[];

  /** Reply target (optional) */
  replyTo?: string;
}
```

#### Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `messageId` | `string` | ❌ | Unique message identifier; a UUID is generated automatically if omitted |
| `channelId` | `string` | ✅ | Channel unique identifier |
| `target` | `object` | ✅ | Message recipient |
| `target.type` | `string` | ✅ | Target type: `user`, `group`, or `channel` |
| `target.id` | `string` | ✅ | Target unique identifier |
| `target.name` | `string` | ❌ | Target display name |
| `content` | `object` | ✅ | Message content |
| `content.text` | `string` | ✅ | Text content of the message |
| `content.format` | `string` | ❌ | Text format: `plain`, `markdown`, or `html`. Defaults to `plain` |
| `media` | `array` | ❌ | Array of media attachments (see [Media](#media)) |
| `replyTo` | `string` | ❌ | ID of the message being replied to |

### Content

Text content and format of the message.

```typescript
interface Content {
  /** Text content */
  text: string;

  /** Content format */
  format?: 'plain' | 'markdown' | 'html';
}
```

#### Format Types

| Format | Description | Example |
|--------|-------------|---------|
| `plain` | Plain text (default) | `"Hello, world!"` |
| `markdown` | Markdown-formatted text | `"**bold** *italic* [link](https://example.com)"` |
| `html` | HTML-formatted text | `"<b>bold</b> <i>italic</i>"` |

#### Examples

##### Plain text

```json
{
  "text": "Hello, this is a test message.",
  "format": "plain"
}
```

##### Markdown

```json
{
  "text": "# Heading\n\nThis is **bold** and *italic* text.\n\n- Item 1\n- Item 2",
  "format": "markdown"
}
```

##### HTML

```json
{
  "text": "<h1>Heading</h1><p>This is <strong>HTML</strong> content.</p>",
  "format": "html"
}
```

### Media

Media file attachment.

```typescript
interface Media {
  /** Media type */
  type: 'image' | 'audio' | 'video' | 'file';

  /** Media file URL */
  url: string;

  /** File MIME type */
  mimeType?: string;

  /** File size in bytes */
  size?: number;

  /** File name */
  filename?: string;

  /** Thumbnail URL (for images and videos) */
  thumbnail?: string;

  /** Duration in seconds (for audio and video) */
  duration?: number;

  /** Image or video dimensions */
  dimensions?: {
    width: number;
    height: number;
  };
}
```

#### Media Types

| Type | Description | Common MIME Types |
|------|-------------|-------------------|
| `image` | Image file | `image/jpeg`, `image/png`, `image/gif`, `image/webp` |
| `audio` | Audio file | `audio/mpeg`, `audio/wav`, `audio/ogg` |
| `video` | Video file | `video/mp4`, `video/webm`, `video/quicktime` |
| `file` | Other file | `application/pdf`, `application/zip`, etc. |

## Message Types

The system supports the following `messageType` values:

| Type | Description | Use Case |
|------|-------------|----------|
| `text` | Plain text message | Regular chat |
| `image` | Image message | Sending images |
| `audio` | Audio message | Voice messages, music |
| `video` | Video message | Video sharing |
| `file` | File message | Documents, archives, etc. |
| `system` | System message | Notifications, alerts |

### Message Type Resolution

The message type is resolved by the following rules (or set explicitly in the request):

1. If the `media` array is present and non-empty:
   - The first media item's `type` determines the message type
   - Example: `media[0].type === 'image'` → `messageType = 'image'`

2. If no media attachments are present:
   - Defaults to `text`

3. System messages:
   - Must be explicitly set to `system` in backend logic

## API Endpoint

### Forward Message to the Hub

**Endpoint:** `POST /api/channel/messages`

**Authentication:** Channel token required in request headers

**Request Headers:**

| Header | Required | Description |
|--------|----------|-------------|
| `X-Channel-Token` | ✅ | Channel access token |
| `X-Channel-ID` | ✅ | Channel ID (used for logging and validation) |
| `Content-Type` | ✅ | Must be `application/json` |

**Success Response:**

```json
{
  "success": true,
  "messageId": "msg_abc123",
  "deliveredAt": "2026-02-13T10:30:00.000Z"
}
```

**Error Response:**

```json
{
  "success": false,
  "error": "Error description",
  "code": "ERROR_CODE"
}
```

**Error Codes:**

| Code | Description |
|------|-------------|
| `UNAUTHORIZED` | Authentication failed — invalid token |
| `INVALID_REQUEST` | Malformed request |
| `CHANNEL_NOT_FOUND` | Channel does not exist |
| `CHANNEL_DISABLED` | Channel is disabled |
| `INVALID_TARGET` | Malformed target |
| `MISSING_CONTENT` | Missing message content |

## Complete Examples

### Example 1: Send a Plain Text Message

**Request:**

```bash
curl -X POST http://localhost:3000/api/channel/messages \
  -H "Content-Type: application/json" \
  -H "X-Channel-Token: wh_2835b6943ab548dda29b2538ca18e1ef" \
  -H "X-Channel-ID: 23ade6a3-b393-4ed3-895c-41420162e334" \
  -d '{
    "channelId": "23ade6a3-b393-4ed3-895c-41420162e334",
    "messageId": "msg_001",
    "target": {
      "type": "user",
      "id": "user_12345",
      "name": "Alice"
    },
    "content": {
      "text": "Hello! This is a test message."
    }
  }'
```

**Response:**

```json
{
  "success": true,
  "messageId": "msg_001",
  "deliveredAt": "2026-02-13T10:30:00.000Z"
}
```

## Notes

### 1. File Size Limits

Recommended limits:
- **Images**: max 5 MB
- **Audio**: max 10 MB
- **Video**: max 50 MB
- **Files**: max 20 MB

### 2. URL Validity

- All media URLs must be publicly accessible HTTP/HTTPS links
- URLs should be long-lived; avoid temporary signed URLs
- CDN or object storage is recommended

### 3. Message ID Generation

- If `messageId` is not provided, the system generates a UUID automatically
- Using meaningful prefixes (e.g. `msg_`, `test_`) is recommended
- Message IDs should be unique within a channel

### 4. Timestamp Format

- Use Unix timestamp in milliseconds
- JavaScript: `Date.now()`
- Python: `int(time.time() * 1000)`

### 5. Character Encoding

- All text content must be UTF-8 encoded
- The JSON request body must be valid UTF-8

### 6. Rate Limiting

Recommended limits:
- Per channel: 100 messages/minute
- Per user: 30 messages/minute
